# AGENTS.md

## Проект

Сайт на чистом `HTML + CSS + ES modules JS` без сборщика.
Тематика: продажа собственного б/у оборудования для торговли, переработки и производства.
Тон сайта: деловой, понятный, продающий, без канцелярщины и без абстрактных маркетинговых фраз.

Главная задача сайта:
- быстро показать реальные позиции;
- дать перейти в карточку конкретного товара;
- дать клиенту сразу связаться по телефону / в мессенджере / через форму.

## Текущая структура

- `index.html` — главная страница.
- `products.html` — страница одного товара.
- `style.css` — весь визуал, адаптив, анимации, модалка, формы.
- `script.js` — данные товаров, рендер, галерея, табы, видео, формы, мобильное меню.
- `assets/products/` — локальные фото, видео, hero, логотип.

## Логика страниц

### Главная

На главной:
- hero-блок должен занимать первый экран;
- ниже идёт сетка товаров;
- по кнопке `Подробнее` всегда открывается `products.html?product=<slug>`;
- блоки `О компании`, `Контакты`, форма, футер идут ниже.

### Страница товара

На `products.html` НЕ должно быть каталога товаров сверху.
Страница товара должна показывать только:
- хлебные крошки;
- заголовок товара;
- галерею;
- цену / краткий оффер;
- табы `Описание / Характеристики`;
- встроенное видео, если оно есть;
- контакты;
- форму;
- футер.

Если пользователь кликает `Подробнее`, он должен видеть только один выбранный товар и всю информацию только по нему.

## Данные товаров

Данные товаров лежат в `script.js` в массиве `products`.

У товара сейчас используется примерно такая структура:
- `slug`
- `name`
- `price`
- `shortSummary`
- `preview`
- `images`
- `description`
- `specs`
- `video`

### Видео

Видео поддерживается в двух форматах:
- обычный файл через `type: "video"` и `src`;
- YouTube через `type: "youtube"` и `embed` либо через `url`, который приводится к embed.

Нельзя ломать:
- встроенное видео в карточке товара;
- открытие видео в модалке;
- поддержку локального `mp4`;
- поддержку `YouTube`.

## Контент и тексты

Очень важно:
- не писать абстрактные, вычурные или «агентские» тексты;
- не писать слишком общие фразы в духе «эффективные решения для бизнеса» без конкретики;
- не писать тексты как для SaaS или IT-продукта;
- не писать канцелярщину и псевдо-премиальный мусор.

Нужно писать так:
- по теме продажи б/у оборудования;
- просто и по делу;
- понятно для клиента;
- с упором на реальный товар, состояние, применение, наличие, цену, характеристики, удобство связи.

Хороший ориентир по стилю:
- «Продаём своё б/у оборудование. Качество гарантируем, обращайтесь в любое время.»

При переписывании текстов:
- сохранять смысл конкретной товарной категории;
- делать тексты клиентскими, а не технически-отчётными;
- краткий оффер рядом с ценой должен быть коротким и продающим;
- полное описание в табе может быть длиннее и предметнее.

## Дизайн

Текущий визуальный вектор:
- тёмная база;
- аккуратные оранжевые и синие акценты;
- крупный hero;
- заметный, но не кричащий логотип;
- карточки с реальными фото;
- плавные анимации появления и hover-эффекты;
- нормальная мобильная версия.

Не делать:
- шаблонный «нейро»-дизайн;
- эмодзи в интерфейсе;
- случайные декоративные стикеры;
- перегруженные градиенты;
- лишние псевдо-3D/стеклянные эффекты;
- мелкий логотип.

## Изображения и медиа

Фото и видео должны быть локальными, если уже скачаны в `assets/products/`.
Предпочтение:
- использовать локальные файлы вместо внешних ссылок;
- не удалять уже скачанные медиа без причины;
- следить, чтобы превью в карточках выглядели нормально в `object-fit: cover`.

Если добавляются новые товары:
- сначала скачать локально фото/видео;
- потом подключать в `script.js`.

## Логотип

Логотип сейчас должен быть своим, аккуратным, тематическим, векторным (`assets/products/logo.svg`).

Нельзя:
- возвращать старый временный логотип;
- подставлять текстовый логотип вместо графического;
- уменьшать логотип до плохо читаемого состояния.

## Формы

Формы должны иметь фронтовую валидацию.
Сейчас важно сохранять:
- проверку имени;
- проверку телефона;
- проверку сообщения;
- маску телефона;
- показ ошибок по полям;
- success-сообщение после отправки.

Не убирать валидацию.

## Мобильное меню

На мобильных:
- меню не должно дёргать страницу при открытии и закрытии;
- нужно сохранять компенсацию ширины скроллбара;
- body lock должен работать аккуратно.

## Что обязательно проверять после правок

После любых изменений проверить:

1. `node --check script.js`
2. главную страницу:
   - hero первый экран;
   - карточки товаров;
   - переход `Подробнее`
   - контакты и форма
3. страницу товара:
   - открывается только один товар;
   - галерея работает;
   - табы работают;
   - видео работает;
   - контакты и форма работают
4. мобильную версию:
   - меню;
   - отсутствие скачков layout;
   - hero;
   - карточки;
   - футер

## Предпочтения по дальнейшей работе

Если что-то меняется:
- сначала сохранять общую логику сайта;
- не ломать простую структуру;
- не превращать сайт в маркетинговый лендинг ради красоты;
- помнить, что это сайт под продажу конкретного б/у оборудования.

Главное правило:
- больше реального товарного смысла;
- всё должно быстро вести клиента к товару и к контакту.
